# GROUNDING

Field- and organization-level constraints that survive runtime, model, and
task changes. Lower-layer files (`AGENTS.md`, runtime overlays, `SKILL.md`,
`plan.md`) may **specialize** these constraints but never relax them.

## Hard Constraints (HC)

- HC1 — No plaintext secrets in repository, prompts, logs, or commits.
- HC2 — No writes outside the repository working tree.
- HC3 — Destructive commands (`rm -rf`, `git push --force`, schema drops,
  package publishes) require explicit human approval, even in
  `accept-execute` mode.
- HC4 — Migrations and irreversible operations may be authored by the agent
  but are never executed by the agent.
- HC5 — Network egress from the agent sandbox is denied by default; allowlist
  is documented in `AGENTS.md`.
- HC6 — Red-class data (see Data Classification) never enters prompts, tool
  arguments, or logs.

## Data Classification

| Class | Examples | Allowed in prompts? | Allowed in logs? |
|---|---|---|---|
| Green | OSS source code, public docs | yes | yes |
| Amber | Internal source, non-PII configs | yes (redact tokens) | redacted |
| Red | Secrets, PII, customer data, license keys | **no** | **no** |

## Enforcement Hooks

Deterministic checks that fail the run if violated:

- `gitleaks` (or equivalent) on every commit.
- Path allowlist enforcement for write operations.
- Invariant scripts under `.agent/policies/`.
- Manifest regression suite (`local-agent-harness`).

> Do not edit this file casually. Changes to `GROUNDING.md` require a
> dedicated PR, review, and a passing manifest regression suite.
